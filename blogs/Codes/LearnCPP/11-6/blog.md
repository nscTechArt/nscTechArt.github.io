### Function templates

---

假如我们想编写一个求两数之间较大值的函数，代码可能是这样的

```c++
int max(int x, int y)
{
	return (x < y) ? y : x;
}
```

虽然在函数调用时可以将不同的值传递到函数中，但参数的类型是固定的，因此函数调用只能传入 `int` 值。这意味着这个函数实际上只适用于整数（以及可以提升为 `int` 的类型）。如果想要实现在double类型的两个数中返回最大值，就需要使用`double`类型作为参数的函数重载。

但是为每个参数类型实现对应的函数重载会造成代码的冗余，也不便于维护。为了解决这个问题，C++为我们提供了函数模版这个功能。

---

#### Introduction to C++ templates

在C++中，模版系统旨在减缓创建能够处理不同数据类型的函数/lei的过程。使用模版时，我们不再需要手动创建一堆几乎相同的函数/类，而是创建一个模版。与通常的定义（需要有明确地指定类型）不同的是，在模版中，我们可以使用一个或多个占位符，占位符用来表明在编写模版时，某种未知的数据类型。

一旦模版定义完成，编译器会使用模版来生成任意数量的重载函数/类，每个函数/类都使用不同的实际类型。最终的结果与我们手动创建重载函数是一样的，但是我们只需要创建和维护一个模版，编译器会帮我们完成剩余的工作。

---

#### Function templates

函数模版是一个类似函数的定义，用于生成一个或者多个重载函数，每个函数都有一组不同的实际类型。当创建函数模版时，我们使用占位符类型，或者说模版类型来代替函数中使用的任何参数类型、返回类型或者稍后要指定的类型。

---

#### Creating a templated max function

我们以下面这段代码为例创建函数模版：

```c++
int max(int x, int y)
{
	return (x < y) ? y : x;
}
```

这段代码中，只是用到了一个数据类型`int`，我们将int替换为一个模版参数`T`

```c++
T max(T x, T y)
{
	return (x < y) ? y : x;
}
```

但是现在编译器还不能编译，因为它完全不知道T是什么，编译器会将这段代码视为一个普通函数，而非函数模版。我们还需要告诉编译器，这就是一个函数模版，其中`T`是模版参数，是任意类型的占位符。

模版参数的声明仅限于后面的函数模版，所以每个函数模版都需要自己的模版参数声明。

```c++
template <typename T> // this is template parameter declaration
T max(T x, T y)
{
	return (x < y) ? y : x;
}
```

在我们的模版参数声明中，template关键字会告诉编译器，我们在创建一个模版。接下来，我们将模版函数会用到的所有模版参数指定在尖括号内。对于每个类型的模版参数，我们会使用关键字`template`或`class`，后面紧接着的是模版参数名字，比如`T`

在这种语境下，关键字`template`和`class`是没有区别的。但是我们最好还是优先使用`template`，以免带来不必要的歧义
